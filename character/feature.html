Feature

Here is some main mechanism this plugin provide

Use InputSystem Support

Built-in character script have virtual methods for new input system to handle, You can get the input action asset in each demo folder

Camera Control

User can easily control camera transform, It use a camera rigging to handle the problem.

Parent children transform structure:
Character -> Camera rigging -> Camera (Attached camera component)

In this form of structure, it has more flexibility to control the camera

Menu Item Quick Setup

Quickly setup an pre-defined character

Camera Vibration

User can switch the vibration behaviour easily, abstract class also provide vibration method to use.

Rotation Handling

Provide freedom of degree, user can lock x axis camera rotation or camera rigging upper limit and lower limit, so it won't rotate over 90 or -90 degree causing problem.

Movement

Provide basic movement method, simply moving the character position, you can choose depending rotation, So it won't only moving in world rotation

Customization

User can create their own character script and inherit built-in classes
